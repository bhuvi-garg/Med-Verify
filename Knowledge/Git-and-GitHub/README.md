# Git & GitHub Basics

Welcome! This mini-course teaches you **Git** (a tool for saving snapshots of your code as you work) and **GitHub** (a website that stores those snapshots online) — skills you'll use in every coding project from here on.

This folder is self-contained and generic — it teaches standard, textbook Git. If you're using these skills on a specific project, that project may have its own document describing any project-specific rules or conventions (a particular commit message format, a specific way of syncing, who's allowed to merge, etc.) — check for one before assuming "the standard way" is exactly how that project works.

## Read these in order

1. **[01 — What is Git and GitHub?](01-what-is-git-and-github.md)** — the big idea, in plain words, with a picture.
2. **[02 — Cloning a Repository](02-cloning-a-repo.md)** — getting an existing project onto your computer.
3. **[03 — Your Everyday Workflow](03-everyday-workflow.md)** — the loop you repeat every time you write code: pull, edit, stage, commit, push.
4. **[04 — Branching & Working With Others](04-branching-and-teamwork.md)** — how multiple people work on the same project at once, Pull Requests, and how GitHub can merge them back together.
5. **[Exercises](exercises.md)** — hands-on practice, using a scratch project of your own.
6. **[Cheat Sheet](cheat-sheet.md)** — one page to keep open while you work.

## The one-sentence version

> Git saves snapshots of your code as you work, and GitHub is a website that stores those snapshots online so you (and others) can get them from anywhere.
